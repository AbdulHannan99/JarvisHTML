📖 Vanilla JS vs jQuery

A quick reference guide used to build this very app. Open app.js alongside this page to follow along.

❓ What is jQuery?

jQuery is a JavaScript library — a collection of pre-written functions that make common tasks shorter to write. It was created in 2006 to solve cross-browser pain points that no longer exist in modern browsers. Today it is still widely used in legacy codebases and is a great teaching tool because the before/after comparison makes DOM concepts very clear.

  • You load it once via a <script> tag — then $ is available everywhere.
  • Every $(...) call returns a jQuery object — a wrapper around one or more DOM elements.
  • jQuery objects have chainable methods: .text().addClass().on() — all in one line.
  • To get the raw DOM element back out, use $el[0].

📌 Rule of Thumb — What we kept Vanilla in this app

Not everything benefits from jQuery. In this project we made deliberate choices:

jQuery
  • Selecting DOM elements
  • Adding / removing CSS classes
  • Reading & writing input values
  • Creating & inserting elements
  • All event listeners
  • Animated scroll
Kept Vanilla JS
  • fetch() + async/await — modern HTTP, jQuery's $.ajax is outdated
  • sessionStorage — no jQuery equivalent
  • sanitize() — uses raw DOM for XSS prevention
  • JSON.stringify / parse — pure JS, no wrapping needed

⚡ Key Differences to Remember

  • .prop() vs .attr() — Use .prop() for boolean DOM properties (disabled, checked). Use .attr() for HTML attributes (placeholder, href).
  • $el[0] — jQuery wraps DOM nodes. To pass a jQuery object to something that needs a raw DOM node (like $.contains()), unwrap with [0].
  • Chaining — jQuery methods return the jQuery object, so you can chain: $el.text(msg).addClass('visible').focus()
  • this inside .on() — In a jQuery event handler, this is the raw DOM element, not a jQuery object. Wrap it if needed: $(this)
  • $(document).ready vs window load — ready fires when HTML is parsed (earlier). load waits for images too. For DOM manipulation, ready is almost always what you want.
